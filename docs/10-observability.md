# 10 · 可观测性：日志 / 崩溃分析 / 监控

> 对应 JD "完善崩溃分析、日志体系、监控"，以及"强大问题定位——能清晰区分是 App、BLE 还是固件的问题"。可观测性是这条"分层定位"能力的基础设施。

## 1. 目标

- **分层结构化日志**：按 BLE 字节流 / 帧 / 命令 / 状态 / OTA / 计算推理 分类，可开关、可导出，便于**两端对齐字节流**与定位问题归属（App vs BLE vs 固件/模拟器）。
- **崩溃与性能采集**：系统级 `MetricKit` 为主，可选第三方。
- **监控指标**：连接/传输/OTA 等关键指标本地埋点，调试面板可视化。
- 全部在**无硬件、模拟器**条件下即可自证。

## 2. 分层日志体系

### 2.1 技术选型
- **`OSLog` / `Logger`（推荐）**：高性能、隐私分级、可用 Console/`log` 抓取；或 **swift-log** 作为门面(便于替换后端)。
- 统一封装 `HRSenseLog`（在 `HRSenseCore` 或独立 `HRSenseLogging` 包），对上层给分类接口。

### 2.2 日志分类（category）

| Category | 内容 | 典型用途 |
| --- | --- | --- |
| `ble.raw` | 原始字节流(hex) | 与模拟器/固件对齐字节 |
| `ble.frame` | 分片/重组/CRC/seq | 定位帧级错误 |
| `ble.conn` | 扫描/连接/断连/重连/MTU | 连接稳定性 |
| `proto.cmd` | 命令/响应/ACK/协商 | 协议交互 |
| `state` | Redux 状态迁移(Action→State) | 复现问题 |
| `ota` | OTA 各阶段/进度/错误 | 升级排障 |
| `compute` / `infer` | 计算/推理耗时与结果 | 性能/正确性 |
| `perf` | 吞吐/丢样/帧率(见 spec 0003) | 波形性能 |

### 2.3 级别与开关
- 级别：`debug / info / notice / error / fault`。
- **运行时可调**：debug 构建默认 verbose；release 默认仅 `notice+`；提供**开发者菜单**切换分类与级别。
- **隐私**：健康数据字段标记为私有(`privacy: .private`)，避免明文进系统日志。

### 2.4 导出与对齐
- 支持**导出诊断包**（一段时间的分类日志 + 关键指标快照），用于问题上报与"两端字节对齐"。
- 约定 App 与模拟器用**同一 hex 打印格式**，便于并排 diff（见 `05` 控制台日志）。

## 3. 崩溃分析与性能

### 3.1 采集
- **MetricKit（首选，系统级、无三方依赖）**：`MXMetricManager` 收集崩溃诊断(`MXCrashDiagnostic`)、卡顿(hang)、CPU/内存/耗电/磁盘等 `MXDiagnostic`。
- **可选第三方**：Crashlytics / Sentry（符号化崩溃、实时上报）。**默认不引入**，如需再评估许可与隐私。

### 3.2 符号化与关联
- 保留 dSYM；崩溃报告与**最近状态/日志**关联(附最近 N 条 `state`/`ble.conn`)，加速定位。

### 3.3 关键性能场景
- 波形高吞吐(spec 0003)：CPU/内存/帧率。
- OTA(07)：耗时/失败。
- 后台 BLE(见 `04` 后台小节)：唤醒/耗电。

## 4. 监控指标（本地埋点 + 调试面板）

| 指标 | 来源 | 用途 |
| --- | --- | --- |
| 连接成功率 / 平均连接耗时 | `ble.conn` | 连接质量 |
| 重连次数 / 重连成功率 | 重连状态机(`04`§5) | 稳定性 |
| 命令超时率 | 命令层(2s×3, `03`§8) | 协议健康 |
| 数据丢样率 / 波形丢块率 | `sampleSeq`/`blockSeq` | 传输质量 |
| 有效吞吐 / 端到端时延 | spec 0003 §3.6 | 波形性能 |
| OTA 成功率 / 平均耗时 | `ota` | 升级质量 |
| 崩溃率 / 卡顿次数 | MetricKit | 稳定性 |

- **调试面板**（DEBUG 构建内置一屏）：实时展示上述指标 + 日志分类开关 + 导出诊断包。
- **模拟器联动**：模拟器可注入故障（丢包/断连/OTA 失败），验证指标能如实反映（自证监控有效）。

## 5. 与架构的衔接

- 日志/埋点作为**横切关注点**：在 `HRSenseData`(BLE/解码)、`HRSenseFeature`(状态/中间件)、`HRSenseSimulatorKit` 各关键路径插桩。
- Redux **Logging Middleware**：统一记录 `Action → State` 迁移（`state` 分类），天然可复现问题序列。
- 指标聚合在一个 `MetricsCollector`（`HRSenseData` 或独立），对 UI 面板与导出提供只读快照。

## 6. 决策（已固化）
- [x] 日志：**OSLog/Logger + 统一分类门面**，运行时可调级别、隐私分级、可导出诊断包。
- [x] 崩溃/性能：**MetricKit 为主**，第三方可选（默认不引入）。
- [x] 监控：**本地埋点 + DEBUG 调试面板**，模拟器故障注入用于自证。
- [x] Redux **Logging Middleware** 记录状态迁移。
