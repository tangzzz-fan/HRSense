# 0003 · 实时波形高吞吐传输与高性能可视化

- **状态**: draft
- **作者**: TBD
- **创建日期**: TBD
- **关联文档**: [../03-ble-gatt-protocol.md](../03-ble-gatt-protocol.md)、[../05-simulator-macos.md](../05-simulator-macos.md)、[../04-app-clean-redux.md](../04-app-clean-redux.md)

> 对应 JD "大数据流(实时波形)吞吐" 核心难点。目标：在 **无硬件、macOS 模拟器** 条件下，建立**高通量波形通道**并能**量化吞吐/时延/丢样**，同时保证 UI 高性能不卡。

## 1. 背景与问题

心率 bpm 是低频小样本；而 **ECG/PPG 波形**是高频（如 128–512 Hz）连续采样，数据量高出 1–2 个数量级。这对 BLE 传输（MTU、连接参数、通知节奏、分片/重组、背压）与端上渲染都是挑战，也是本项目相对 JD 的**最大差距**。

## 2. 目标 / 非目标

### 目标
- 定义**波形数据通道**（协议层）：批量、高效、可检测丢样。
- 给出 **BLE 吞吐优化**清单与本项目采用的固化策略。
- 定义 **背压/降采样**策略（传输侧与 UI 侧）。
- 定义**高性能可视化**方案（实时波形自绘）。
- 模拟器可产生合成波形并**量化吞吐/时延/丢样**，App 侧可统计与展示。

### 非目标
- 不做真实医疗级 ECG 算法/精度（合成信号即可验证链路）。
- 不定死采样率/位宽的业务取值（给可配置默认）。

## 3. 方案

### 3.1 波形数据模型（L4 扩展，见 `03` §6）

- 新增 `DataKind = 波形(0x02)`（见 `03` §6.2 扩展）。
- **批量帧**：一帧承载**一个采样块**（N 个采样点），而非单点，摊薄帧头开销。
- 字段（TLV，见 `03` 字段字典扩展）：

| Tag | 字段 | 类型 | 说明 |
| --- | --- | --- | --- |
| `0x10` | waveformType | u8 | 1=ECG, 2=PPG（可扩展）|
| `0x11` | sampleRateHz | u16 | 采样率 |
| `0x12` | blockSeq | u32 | 采样块序号（连续性/丢块检测）|
| `0x13` | startTimestampMs | u32 | 块首采样时间（相对 t0，见 `03` §6.2.1）|
| `0x14` | sampleBits | u8 | 每采样位宽（如 12/16）|
| `0x15` | samples | i16[] | 采样点数组（小端；按位宽打包）|

- **连续性**：`blockSeq` 递增 + 块内采样点数固定 → App 可精确算出丢块并做补偿/标记。

### 3.2 传输通道与吞吐优化（固化策略）

| 手段 | 策略 | 说明 |
| --- | --- | --- |
| **MTU 最大化** | 连接后请求最大 MTU，按 `min(MTU)-ATT头-帧头` 决定块负载 | 减少分片、提升有效载荷比 |
| **批量打包** | 每个 notify 尽量塞满一个采样块 | 摊薄帧头/通知开销 |
| **通知方式** | 波形走 **Notify**（无需逐包确认）；best-effort + `blockSeq` 丢块统计 | 实时性优先（与 `03` §8 数据通道策略一致）|
| **连接参数** | 记录/说明 iOS 对 connection interval 的限制（App 只能"请求"，由系统协商）| 面试可讲清 iOS 限制边界 |
| **写方向大流量** | 若上行大流量用 **Write Without Response** + 窗口流控（复用 OTA 思路，见 `07`）| 本项目波形是下行为主 |
| **解码位置** | 分片重组 + 解包在**后台队列**，避免阻塞主线程 | 见 `04` §4 |

> 固化：波形通道 = **Notify + 批量块 + best-effort + blockSeq 丢块检测**；MTU 与块大小在连接时动态计算。

### 3.3 背压与降采样

- **传输侧**：模拟器可配置采样率；App 若消费不过来，**丢弃最旧块**（环形缓冲）并计入丢样统计，绝不无限堆积内存。
- **计算侧**：原始波形进**有界环形缓冲**（如保留最近 N 秒）供算法/存储取用。
- **UI 侧**：渲染做 **min/max 抽样降采样**（保形），把每屏像素列映射到区间的极值，避免逐点绘制。

### 3.4 高性能可视化（对应 JD "高性能图表"）

- **分层选型**：
  - 趋势/HRV/统计图 → **Swift Charts**（够用、省事）。
  - **实时波形** → **自绘**：`Canvas`（TimelineView 驱动）或 `CAShapeLayer`/Metal；**环形缓冲 + 固定窗口滚动**。
- **要点（固化）**：
  - 刷新预算：波形 UI 刷新 **≤60fps**、心率/趋势 **≤2Hz**（见 `04` §8.3）。
  - **降采样绘制**：min/max 抽样到屏宽像素列数。
  - 复用/离屏：避免每帧重建路径；只追加新样本、滚动窗口。
  - 原始全量仍进计算/存储管线，不受 UI 降采样影响。
- **落点**：`04` "可视化策略"小节引用本节。

### 3.5 模拟器扩展（`05`）

- **波形发生器**：合成 ECG（PQRST 模板叠加基线漂移/噪声）与 PPG（脉搏波），采样率/幅度/噪声可配。
- **吞吐/丢样面板**：显示发送块速率、有效码率(bytes/s)、模拟丢块。
- **故障注入**：高频下丢块、突发延迟、块乱序、块内截断（验证 App 重组/丢块统计/UI 稳定）。

### 3.6 度量指标（App 侧统计，联动 `docs/10` 可观测性）

| 指标 | 定义 |
| --- | --- |
| 有效吞吐 | 每秒成功解出的采样点数 / bytes |
| 端到端时延 | 块 `startTimestampMs` → 上屏时间 |
| 丢块率 | `blockSeq` 缺口 / 总块数 |
| UI 帧率 | 波形视图实际渲染帧率 |
| 内存占用 | 环形缓冲上限内 |

## 4. 备选方案与取舍
- **逐点单帧下发（未采用）**：帧头开销大、通知次数爆炸、吞吐低。→ 用**批量块**。
- **可靠重传波形（未采用）**：实时性差、复杂度高。→ **best-effort + 丢块统计**，必要时应用层做插值/标记。
- **Swift Charts 画波形（未采用为主渲染）**：高频易卡。→ 波形自绘、Charts 只做低频图。

## 5. 影响面
- `03`：新增波形 DataKind + 字段 + 吞吐章节。
- `05`：波形发生器 + 吞吐/丢样面板 + 相关故障。
- `04`：可视化策略小节；`HRSenseData` 解码/环形缓冲；指标埋点。
- `docs/10`：吞吐/丢样/帧率指标纳入监控。

## 6. 测试策略
- **协议单测**：波形块编解码、`blockSeq` 丢块检测、块内截断处理。
- **吞吐基准**：模拟器阶梯提升采样率，记录有效吞吐/丢块/时延曲线（Mac↔iOS 真机）。
- **UI 压测**：高采样率下波形视图帧率与内存稳定性。
- **故障**：丢块/乱序/延迟下 UI 不崩、统计准确。

## 7. 决策与开放问题（已固化）
- [x] 波形通道 = **Notify + 批量采样块 + best-effort + blockSeq 丢块检测**。
- [x] 块负载按 **MTU 动态计算**；解码在后台队列。
- [x] 背压 = **丢最旧块 + 计入统计**；UI = **min/max 降采样自绘**。
- [x] 可视化：**波形自绘 + Charts 低频图**分层。
- [ ] 默认采样率/位宽的业务取值（给可配置默认，业务确定后收敛）。
